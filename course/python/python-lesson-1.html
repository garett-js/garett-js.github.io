Основы Python и настройка окружения

Что будем изучать

Программирование — это процесс создания инструкций (программного кода), по которым компьютер или другое устройство выполняет определённые задачи и операции.

Чтобы машина правильно поняла наши инструкции, программисты используют специальные языки программирования, на которых пишут код.

В основе программирования лежит составление алгоритмов — последовательностей действий, которые приводят к нужному результату. Программист переводит эти алгоритмы на язык, понятный компьютеру, используя правила синтаксиса выбранного языка программирования.

Компьютеры работают на машинном коде — последовательности из нулей и единиц. Такой код совершенно неудобен для восприятия человеком, ведь даже самое простое действие требует длинного набора бинарных команд. Например, чтобы отобразить на экране фразу «Привет, Мир!», в машинном коде это может выглядеть примерно так:

Попробовать написать это вручную — задача почти невозможная для человека, особенно без специальных знаний. Именно поэтому и были созданы языки программирования высокого уровня. Они позволяют описывать действия с использованием слов и понятий, близких обычному человеческому мышлению.

Например, та же программа на языке Python выглядит просто и понятно:

Python
print("Привет, Мир!")

Такой код может прочитать даже тот, кто никогда раньше не сталкивался с программированием. Программные языки служат своего рода переводчиком между человеком и машиной — они превращают понятные нам слова в команды, которые способен выполнить компьютер.

Сейчас технологии шагнули ещё дальше — ИИ (Искусственный Интеллект) умеет интерпретировать команды, написанные обычным человеческим языком. Однако на текущем этапе всё ещё крайне важно разбираться в основах программирования, чтобы понимать, что именно генерирует искусственный интеллект и как это работает под капотом.

Язык программирования Python

Язык Python - это высокоуровневый язык программирования общего назначения с динамической типизацией и автоматическим управлением памятью.

Используется для создания веб-приложений, анализа данных, машинного обучения, автоматизации и многого другого.

Язык Python разработал голландский программист Гвидо Ван Россум (Guido van Rossum) в 1991 году. Гвидо был фанатом британского комедийного сериала «Monty Python’s Flying Circus», откуда и пришло название языка. (Почему логотип потом стала змея, не понятно)

Преимущества Python

  • Простота и читаемость кода
  • Огромное количество библиотек и фреймворков
  • Большое и активное сообщество
  • Широкое применение в различных областях (создавать сайты, игры, анализировать данные, тренировать нейросети и многое другое )
  • Подходит для новичков

Недостатки Python

  • Низкая скорость выполнения по сравнению с другими компилируемыми языками
  • Динамическая типизация языка, это боль при написании сложных программ.

Python Интерпретатор

Языки программирования делятся на два вида

  1. компилируемые
  2. интерпретируемые

Программу на компилируемом языке (Например, C++ или Dart) перед выполнением нужно проверить на наличие синтаксических ошибок и уже после этого перевести в понятную для компьютера форму — машинный код. Это делает специальная программа, которая называется компилятором.

Программа на интерпретируемом языке (Например Python, JavaScript или Ruby), не переводится в машинный код. Вместо этого специальная программа, которая называется интерпретатором, идет по коду, анализирует его и выполняет каждую отдельную команду строчка за строчкой.

Существуют языки, совмещающие оба подхода (Например C# или Java). Исходной код программы сначала компилируется в промежуточный код (байт-код), а уже потом, во время выполнения, переводится в машинный код.

Интерпретатор

Для этого нужен специальный инструмент — интерпретатор. Он переводит написанный вами код с языка, понятного человеку, на язык, понятный компьютеру, просматривая его построчно. Этот процесс называется интерпретацией.

Если интерпретация проходит успешно, программа запускается. Когда компьютер выполняет вашу программу, он последовательно исполняет каждую из ваших инструкций.

Кавычки

Всё что записано в двойные или одинарные кавычки является обычным текстом / строкой

Основные литералы: числа и строки

Независимо от сложности, все программы выполняют операции с числами, строками и другими значениями. Эти значения называются литералами.

Целые числа Integer

Мы используем целые числа, например, для подсчета вещей в реальном мире.

Например 0, 1, 2, 10, 11, 100

Строки String

Строка это последовательность любых символов заключенных в двойные или одинарные кавычки.

Например "text", 'Я изучаю Python', "123456", 'e-mail@yandex.ru', " "

Строки могут включать буквы, цифры, пробелы и другие символы.

Комментарии

Комментарии в коде нужны, чтобы объяснять, что делает каждая часть программы. Они помогают понять, как работает код, и облегчают его чтение. Когда кто-то другой (или даже вы сами позже) будет смотреть на ваш код, комментарии подскажут, что именно вы хотели сделать.

При интерпретации комментарии игнорируются, не влияя на выполнение программы и её размер. В языке Python существуют два вида комментариев:

Однострочные комментарии #

Всё, что написано после # до конца строки, считается комментарием

Однострочный комментарий
# Это однострочный комментарий
print("Привет, Python!") # Вывести на экран строку

Многострочные комментарии """ """ или ''' '''

Такие комментарии могут занимать несколько строк, заключенные между тройными двойными или тройными одинарными кавычками.

Многострочный комментарий
"""Это
Многострочный
Комментарий
"""
print("Привет, Python!")

Арифметические операции → бинарные операторы

В Python существует несколько основных арифметических операторов:

  • Сложение +
  • Вычитание -
  • Умножение *
  • Деление / - всегда возвращает float
  • Целочисленное деление //
  • Остаток от деления %
  • Возведение в степень **

Эти операторы являются бинарными, то есть они оперируют двумя значениями (операндами). Операнд - это значение или переменная, к которой применяется оператор.

Операторы и Операнды

В выражении 1 + 3 числа 1 и 3 являются операндами, а + является оператором.

Примеры операций
print(12 + 42) # 54
print(10 - 70) # -60
print(21 * 30) # 630
print(20 / 20) # 1.0
print(18 % 10) # 8
print(2 ** 3) # 8
print(20 // 6) # 3

Сложные выражения

Арифметические операции можно комбинировать для создания сложных выражений:

Сложное выражение
print(1 + 3 * 4 - 2) # печатает 11

Порядок вычислений совпадает с обычными правилами арифметики. Сначала скобки, потом возведение в степень, затем умножение/деление, и в конце сложение/вычитание.

Выражение со скобками
print((1 + 3) * (4 - 2)) # печатает 8

Скобки могут быть вложенными

Вложенные скобки
print(((54 // 10) % 10) * 2) # печатает 10

Ошибки в коде и их исправление

Ошибки или баги в программах бывают двух видов:

  1. Ошибки синтаксиса (ошибки не дают запуститься программе)
  2. Ошибки исполнения (ошибки возникают во время работы программы)

Рассмотрим пример с ошибками синтаксиса и то как их исправить.

Ошибки синтаксиса

Когда код не запускается, это означает, что где-то возникла синтаксическая ошибка.

Запустите этот код, что вы увидите?

Код с ошибкой
print("Сегодня дождь

При запуске программы появится сообщение об ошибке:
SyntaxError: unterminated string literal (detected at line 1)

Это означает, что интерпретатор дошел до конца строки, но так и не нашел закрывающую кавычку для строки.

В редакторе VSCode ошибка будет выделена красным цветом.

Если навести мышкой на ошибку, то появится подсказка с описанием ошибки.

Исправим этот код и запустим снова на выполнение

Исправленный код
print("Сегодня дождь")

Теперь код выполнится без ошибок.

Стиль программирования на языке Python

  • Имена функций и переменных должны быть написаны в стиле snake_case (слова разделяются нижним подчеркиванием).
  • Каждое выражение должно находиться на отдельной строке.
  • Отступ в 4 пробела используется для обозначения блока кода. Не используйте символ табуляции для отступов.
  • Между функциями и классами верхнего уровня оставляйте две пустые строки.
  • Старайтесь, чтобы длина строки не превышала 79 символов.

The Zen of Python

Разработчики языка Python придерживаются определённой философии программирования, называемой «The Zen of Python». Её текст выдаётся интерпретатором Python по команде import this (работает один раз за сессию). Автором этой философии считается Тим Петерс (Tim Peters).

В оригинале

  1. Beautiful is better than ugly;
  2. Explicit is better than implicit;
  3. Simple is better than complex;
  4. Complex is better than complicated;
  5. Flat is better than nested;
  6. Sparse is better than dense;
  7. Readability counts;
  8. Special cases aren't special enough to break the rules;
  9. Although practicality beats purity;
  10. Errors should never pass silently;
  11. Unless explicitly silenced;
  12. In the face of ambiguity, refuse the temptation to guess;
  13. There should be one — and preferably only one — obvious way to do it;
  14. Although that way may not be obvious at first unless you're Dutch;
  15. Now is better than never;
  16. Although never is often better than *right* now;
  17. If the implementation is hard to explain, it's a bad idea;
  18. If the implementation is easy to explain, it may be a good idea;
  19. Namespaces are one honking great idea — let's do more of those!
