Урок 10: Словари (Map)

Словари (Map) в Dart

Проблема хранения связанных данных в списках

Возьмём обычный школьный журнал. В нем есть список учеников, и для каждого ученика оценки по разным предметам. Как можно связать ученика с его оценками, используя списки и переменные?

Пробуем хранить данные в списках:

Dart - Поиск оценок в списках

void main() {
  List students = ['Аня', 'Боря', 'Наташа', 'Глеб', 'Даша'];
  List> studentGrades = [
    [5, 4, 5], // Оценки Ани по Математике, Физике, Информатике
    [3, 5, 4], // Оценки Бори
    [5, 5, 5], // Оценки Наташи
    [4, 3, 4], // Оценки Глеба
    [5, 4, 3], // Оценки Даши
  ];

  String student = 'Наташа';

  // Узнать оценки ученика
  int indexOfStudent = -1; // Переменная для хранения индекса
  for (int i = 0; i < students.length; i++) {
    if (students[i] == student) {
      indexOfStudent = i;
      break; // Если нашли, то выходим из цикла
    }
  }

  if (indexOfStudent != -1) {
    print('$student: ${studentGrades[indexOfStudent]}');
  } else {
    print('Ученик $student не найден.');
  }
}

Результат: Наташа: [5, 5, 5]

Почему это "плохо"?

На первый взгляд это работает. Но давайте подумаем о производительности и удобстве.

  1. Производительность (Скорость поиска): Когда мы ищем "Наташу", нам приходится перебирать весь список `students` элемент за элементом. Если учеников 5, это быстро, а если 5 000 000? Это очень долго и не производительно!
  2. Связь данных: Приходится поддерживать два отдельных списка, где соответствие держится только за счет одинаковых индексов. Это хрупко и подвержено ошибкам.

Введение в Map (Словари)

Сейчас мы погрузимся в одну из самых мощных и часто используемых структур данных – Map, или словари (хеши, карты).

Словари (Map)

Map это объект, который представляет собой коллекцию пар ключ-значение. Представьте себе Телефонные Контакты: у каждого телефона (уникального ключа) есть свое определение (значение). Ключ уникален, и по нему вы можете быстро найти соответствующее значение.

Dart - Пример телефонной книги

void main() {
  // Представьте себе телефонную книгу: 
  // Имя (ключ) -> Номер телефона (значение)
  Map phoneBook = {
    'Маша': '+7 911 987 65 43',
    'Артём': '+7 922 555 11 22',
  };
    
  // Получаем значение по ключу
  print('Телефон Маши: ${phoneBook['Маша']}'); 
    
  // Обновляем номер Маши
  phoneBook['Маша'] = '+7 900 777 88 99';
  print('Новый телефон Маши: ${phoneBook['Маша']}');
  
  // Добавляем новый контакт
  phoneBook['Юля'] = '+7 933 444 00 11'; 
  print('Телефон Юли: ${phoneBook['Юля']}');
  
  // Если нет значения в словаре, то будет null
  print('Телефон Джигана: ${phoneBook['Джиган']}');

  print('Вся телефонная книга: $phoneBook');
}

Производительность и Хеш-таблицы

Очень быстрый поиск, добавление и удаление данных.

В отличие от списков (List), Map позволяет найти нужное значение практически мгновенно! Это называется O(1) – Константное время. Время поиска не зависит от количества элементов.

Map в Dart реализован на основе хеш-таблиц. При добавлении пары "ключ-значение", Dart вычисляет **хеш-код (hashCode)** ключа, находит на его основе индекс во внутреннем массиве и помещает туда данные. Это обеспечивает мгновенный доступ.

Вложенные Map: Map внутри Map

Представьте, что нужно хранить не просто один кусок информации, связанный с ключом, а целую группу связанных данных. В таком случае используются вложенные Map. Это когда значением одной Map является другая Map.

Допустим, у нас есть интернет-магазин, и мы хотим хранить информацию о различных товарах. Для каждого товара нам нужен артикул - уникальный идентификатор (ID), а затем для этого ID набор характеристик: название, цена, количество на складе и категория.

Dart - Каталог товаров

void main() {
  Map> products = {
    '123456': {
      'name': 'iPhone 20',
      'price': 170000,
      'stock': 15,
      'category': 'Электроника',
    },
    '987654': {
      'name': 'Механическая клавиатура FlutterSwitch',
      'price': 40000,
      'stock': 50,
      'category': 'Периферия',
    },
    '741258': {
      'name': 'Игровая мышь GigaClick',
      'price': 5000,
      'stock': 30,
      'category': 'Периферия',
    },
  };

  // Доступ к данным
  Map? phone = products['123456'];
  if (phone != null) {
    print('Название: ${phone['name']}');
  }

  // Обновление
  products['123456']!['stock'] = 14;

  // Добавление
  products['963258'] = {
    'name': 'Монитор UltraView 120"',
    'price': 100000,
    'stock': 10,
    'category': 'Электроника',
  };
}

Где это применяется?

  1. JSON-данные (очень часто!): Стандарт для обмена данными в вебе через API.
  2. Файлы с настройками.
  3. Базы данных "ключ-значение" (NoSQL).
  4. Сложные объекты без создания классов.
